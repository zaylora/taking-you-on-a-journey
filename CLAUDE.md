# 依赖优先原则

实现任何功能时，必须按以下顺序决策：

1. **优先寻找成熟开源依赖**：前端特效（粒子、动画、图表等）、算法（排序、加密、数据处理等）、工具函数等，先搜索是否有广泛使用的开源库或框架。
2. **禁止盲目手写**：不得跳过调研直接手写实现，即使实现看起来"很简单"。
3. **手写作为最后手段**：只有在以下情况下才手写实现：
   - 经过调研，确实不存在合适的开源方案
   - 现有依赖包体积过大或存在安全风险
   - 项目有特殊约束（离线、许可证限制等）
   - 手写时必须明确说明"未找到合适依赖，选择手写，原因：……"

# 改动记录规则

每次完成任务或实现改动时，**必须**在 `plan/` 目录下创建对应的记录文档：

## 记录格式

1. **文件夹结构**：`plan/YYYYMMDD_<任务简述>/`
   - 例如：`plan/20260626_add_budget_feature/`

2. **文档内容**（在文件夹内创建 `README.md`）：
   - **任务目标**：这次改动要解决什么问题
   - **改动文件**：列出修改/新增的文件清单
   - **改动详情**：具体改动了哪些内容、为什么这样改
   - **测试结果**：改动后验证的结果
   - **相关讨论**：重要的设计决策或考虑

## 示例结构

```
plan/
├── 20260626_add_budget_feature/
│   └── README.md
├── 20260627_fix_ui_bug/
│   └── README.md
└── ...
```
